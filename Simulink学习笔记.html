第 1 课 · 信号基础

t01_signal_basics

认识 Simulink 的三大基本要素：模块 Block、 信号线 Signal、 仿真运行 Simulation

🔵 Sine Wave → 🟡 Gain ×2 → 🟢 Scope
add_block('simulink/Sources/Sine Wave', ...) % 拖模块 add_line(mdl, 'Sine Wave/1', 'Gain/1') % 连线 set_param(..., 'Amplitude', '3') % 改参数 sim(mdl) % 运行
💡 核心直觉：振幅 A=3 的正弦波经 Gain=2 放大 → Scope 看到振幅 6 的波形。模块 = 乐高积木，连线 = 流水管，仿真 = 打开水龙头。
参数叫什么控制什么
Amplitude振幅波 "有多高"
Frequency角频率波 "有多密"（ω 越大振得越快）
Gain增益信号放大/缩小的倍数

第 2 课 · 数学运算与信号路由

t02_math_blocks

学会把多路信号组合运算，以及 Mux/Demux 的 "捆线拆线"。

Sine1 (ω=1) ↘ Sum + → Product ×5 → Scope1
Sine2 (ω=3) ↗ Constant=5 ↑
🧠 物理直觉：两列正弦波相加 = 叠加原理（傅里叶思想）。乘以常量 = 放大器旋钮。Mux/Demux 不改变信号内容——只是把 3 根电线捆成一束再拆开，就像网线里的 8 根彩色细线。
模块作用物理类比
Sum信号相加/相减两列声波叠加
Product信号相乘放大器增益旋钮
Mux多路信号合并为总线把电线捆成一束
Demux总线拆回多路信号把线束解开还原

第 3 课 · 一阶系统与传递函数

t03_first_order_sys

从 "摆积木" 进阶到 理解系统本身的动态特性。

一阶系统标准形式：G(s) = K / (τs + 1) K — 稳态增益（最终放大倍数） τ — 时间常数（惯性/惰性，越大越慢）
🫖 生活例子：一杯热水放在桌上降温。水温不会瞬间降到室温，而是慢慢逼近——这就是一阶系统的行为。τ 越小 = 杯子越小散热越快；K = 最终温度和初始温度的比值。

教程同时比较了 τ = 0.5 / 2 / 5 三个系统对同一阶跃信号的响应：

τ 值响应速度物理想象
0.5⚡ 最快小杯子，热水凉得快
2.0🐢 中等中等杯子
5.0🐌 最慢大缸，凉得慢
✅ 学会标志：能回答 "τ=0.5 和 τ=5 谁先到稳态？"（τ 越小越快）；"K=3 输入阶跃 0→1，输出最终稳定在几？"（3×1=3）
🎢

第 4 课 · 二阶系统与阻尼特性

t04_second_order_sys

一阶系统的自然延伸——引入 阻尼比 ζ 和 超调量 的概念。

二阶系统标准形式：G(s) = ωₙ² / (s² + 2ζωₙs + ωₙ²) ωₙ — 自然频率（系统 "想振多快"） ζ — 阻尼比（系统 "被抑制多狠"）
阻尼比 ζ行为类比
ζ = 0纯振荡，永远不停理想弹簧（无摩擦）
0 < ζ < 1衰减振荡，有超调汽车过减速带后晃几下
ζ = 1临界阻尼，无超调最快刚好不晃的门
ζ > 1过阻尼，无超调但慢门被阻尼器拉着慢慢关
🔗 和 PID 的联系：ζ 太小 → 超调严重 → 需要 PID 的 D 参数来 "增加阻尼"，抑制超调。这就是为什么第 4 课是第 5 课的前置知识。
🎯

第 5 课 · PID 反馈控制

t05_pid_control

经典控制的核心——让输出自动追着目标跑。

PID 公式：u(t) = Kp·e(t) + Ki·∫e(t)dt + Kd·de(t)/dt e(t) = 设定值 − 实际输出（误差） Kp — 比例：有误差立刻反应（但可能有残余） Ki — 积分：累积误差，消除残余（过大会振荡） Kd — 微分：预测趋势，抑制超调（怕噪声）
设定值 → 误差 Error → PID 控制器 → 被控对象 Plant → 输出
                                              ←────── 反馈回路 ──────←
参数增大效果副作用
Kp ↑响应更快、更猛超调增大、可能振荡
Ki ↑消除稳态误差过大会剧烈振荡
Kd ↑抑制超调、更 "稳"放大高频噪声
🧪 动手实验建议：① 只保留 P（I=0, D=0）→ 看稳态误差；② 加大 I → 看误差消除过程振荡；③ 加大 D → 看超调被抑制；④ I 太大 → 看积分饱和现象。
📂

第 6 课 · 信号导入导出

t06_sources_and_sinks

打通 MATLAB ↔ Simulink 数据交换的通道——让 Simulink 不再是一个封闭沙盒。

方向模块做什么
📥 进口From Workspace把 MATLAB 变量/实验数据喂进 Simulink
📤 出口To Workspace仿真结果 → MATLAB 变量（可做 FFT、统计等）
📤 出口To File仿真结果 → .mat 文件持久化保存
👀 监视Display实时显示当前信号数值（调试神器）
👀 监视Scope看波形（只能眼看，不能处理数据）
MATLAB 变量 → From Workspace → 你的模型 → To Workspace → MATLAB 变量
% 导出后可以做 Scope 做不到的事： max(saved_data.Data) % 求最大值 fft(saved_data.Data) % 频谱分析 mean(saved_data.Data) % 求均值
🌟 一句话：From Workspace = 进口，To Workspace/To File = 出口。学会这扇门，Simulink 就和整个 MATLAB 生态打通了——读取真实数据 → 仿真 → 导出分析 → 写论文，一条龙。

还学了四种信号源：Ramp（线性斜升 ↗）、Pulse（方波脉冲 ⊓⊓）、From Workspace（自定义）、Signal Builder（鼠标画波形）。
