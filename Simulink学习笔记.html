第〇课：什么是 Simulink？为什么要学？

一句话解释

Simulink 是 MATLAB 的"图形化编程"插件。你不用写代码，而是拖拽方块、连线就能搭建一个动态系统模型，然后点"运行"看它怎么动。

类比：如果 MATLAB 是计算器 + 编程语言，那 Simulink 就是电子版的乐高积木。 你从工具箱里拿出"正弦波发生器"、"加法器"、"示波器"这些积木块， 用线把它们连起来，就搭好了一个信号处理流水线。点运行，示波器上就能看到波形。

为什么控制工程师必须学 Simulink？

  • 直观：画图比写代码容易理解。看一个框图就知道信号怎么流，不用读几百行 C。
  • 安全：在真实电机上试 PID 参数可能烧硬件；在 Simulink 里随便试，不会坏。
  • 效率：Simulink 可以一键生成 C 代码部署到 STM32 单片机——这就是"基于模型的设计(MBD)"。
  • 工业标配：特斯拉、大疆、西门子……所有做控制的公司都在用 Simulink。

这套教程适合谁？

  • 学过一点 MATLAB 基础（会定义变量、画 plot 就行）
  • 对控制工程感兴趣，但没系统学过
  • 想从"能跑就行"进阶到"知道为什么这样跑"
学习建议：不要光看！打开 MATLAB，把每个 tXX_xxx.m 脚本跑一遍。 改改参数，看看图怎么变。控制工程是"做"会的，不是"看"会的。

01

信号基础 — 认识模块、信号线、仿真

t01_signal_basics.m

这课在干什么？

搭建你人生中第一个 Simulink 模型：正弦波发生器 → 增益放大器 → 示波器。理解 Simulink 最核心的三个概念：模块(Block)、信号线(Signal Line)、仿真(Simulation)。

核心概念拆解

1. 模块 (Block) — Simulink 的一切都是模块。信号源（Sine Wave）、数学运算（Gain）、输出显示（Scope），每个模块有输入端口和输出端口，像一个有进水口和出水口的水箱。

2. 信号线 (Signal Line) — 两个模块之间的箭头连线，代表"数据从 A 流向 B"。在 Simulink 中信号是有方向的，只能从输出口（>）流到输入口（<）。

3. 仿真 (Simulation) — 按下绿色三角按钮或执行 sim(mdl)，MATLAB 就开始解微分方程，一步步算出每个时间点的信号值。你可以把仿真理解为"快进播放"——0.1 秒内算出系统 10 秒后的状态。

代码是怎么自动搭模型的？

教程中的 .m 脚本用三行核心命令自动搭模型，不需要你手动拖拽：

add_block('simulink/Sources/Sine Wave', [mdl '/Sine1']) ← 放一个正弦波模块 add_line(mdl, 'Sine1/1', 'Gain/1') ← 用线连接两个模块 set_param([mdl '/Gain'], 'Gain', '3') ← 设置增益值为 3 sim(mdl) ← 运行仿真

动手实验

  • 把 Gain 从 3 改成 10，看 Scope 波形幅值变大 10 倍
  • 把 Sine Wave 的频率从 1 改成 5，看波形变密
  • 双击 Scope 模块，用工具栏放大/缩小波形
记住：Simulink 模型 = 模块 + 连线。模块做计算，连线传数据。仿真就是让这个"流水线"运转起来。
02

数学运算与信号路由 — Sum、Product、Mux、Demux

t02_math_blocks.m

这课在干什么？

学四个最常用的运算和路由模块：加法器(Sum)、乘法器(Product)、信号合并(Mux)、信号拆分(Demux)。用两列不同频率的正弦波叠加，直观感受"信号的代数运算"。

逐个模块讲清楚

Sum（加法器） — 把多个输入信号加在一起。可以设置每个输入端口的符号（+ 或 -），图标上会显示 |++ 表示两个正输入。比如你要做 PID 控制，就需要一个 Sum 把 P、I、D 三项输出加起来。

Product（乘法器） — 把多个输入信号乘在一起。注意：Sum 和 Product 看起来差不多（都是圆形），但功能完全不同。乘法在控制里很常见（增益本质上就是乘法）。

Mux（多路复用器） — 把多根独立的信号线"绑成"一根粗线（向量信号）。比如你有位移、速度、加速度三路信号，用 Mux 绑在一起，一根线传给 Scope 的三个端口。

Demux（多路分解器） — Mux 的逆操作。把一根向量信号线"拆开"成多根独立线。Mux 和 Demux 配对使用，就像把几根电线捆成一根电缆传输，到目的地再拆开。

类比：Mux = 把三杯不同的饮料倒进一根三腔吸管里（每腔独立），Demux = 吸管另一端又把三腔拆回三个杯子。信号在传输过程中没混在一起。
常见错误：新手容易把 Mux 和 Sum 搞混。Mux 只改变信号的组织形式（打包/拆包），不改变信号的值。Sum 会真正做加法运算改变信号值。
03

一阶系统 — 传递函数与时间常数

t03_first_order_sys.m

什么是"一阶系统"？

含有一个储能元件的系统。比如一杯热水降温（热能存储）、一个 RC 电路充电（电能存储）。它的数学模型是：

G(s) = K / (τs + 1)

其中 K 是稳态增益（输入 1 最终输出几），τ 是时间常数（系统"惯性"有多大的度量）。

时间常数 τ：控制工程的第一个重要概念

给系统一个阶跃输入（突然从 0 跳到 1），输出不会瞬间跟上——它会"慢慢爬"到稳态值。τ 的意义是：经过 τ 秒，输出达到稳态值的 63.2%。再过 τ 秒，达到 86.5%。再过 τ，达到 95%。

生活类比：你往一杯热水里加冰块——水温不会瞬间变凉，而是慢慢逼近室温。 τ=0.5 相当于小杯子（散热快，凉得快），τ=5 相当于大暖瓶（散热慢，凉得慢）。 控制系统里 τ 越小 = 响应越快 = 越难控制（惯性小，容易抖）。

这课演示了什么？

用 Simulink 的 Transfer Fcn 模块创建三个不同 τ（0.5、2、5）的一阶系统，同时输入阶跃信号，对比它们的响应速度。你会直观看到：τ 越小线条越陡（响应快），τ 越大线条越平（响应慢）。

核心结论：τ 是系统"惰性"的标尺。控制器的任务就是"克服惰性"让系统按你的意愿运动——但惰性越大越费劲。
04

二阶系统 — 阻尼比、超调量与振荡

t04_second_order_sys.m

什么是"二阶系统"？

含有两个储能元件的系统。比如质量-弹簧-阻尼器（动能 + 弹性势能）、RLC 电路（电场能 + 磁场能）。二阶系统最有趣的地方是它会振荡。

G(s) = ωn² / (s² + 2ζωns + ωn²)

三个关键参数

  • ωn（自然频率）：系统"喜欢"以多快的频率振荡。单位 rad/s。
  • ζ（阻尼比）：振荡衰减的快慢。这是最重要的参数！
  • 超调量：输出超过目标的最大百分比。ζ 越小超调越大。

阻尼比 ζ 的四个区间——这课的核心

ζ = 0（无阻尼）：纯振荡，永不衰减。现实中不存在（总有摩擦），但理论上很重要。

0 < ζ < 1（欠阻尼）：会振荡但逐渐衰减。这是最常见的情况（汽车的悬挂、机器人的关节）。ζ = 0.2 时超调约 50%，ζ = 0.7 时超调约 5%——工程上一般取 ζ=0.7 作为"最优"。

ζ = 1（临界阻尼）：刚好不振荡，且上升最快。理论上的"最优"——既没有超调的烦恼，也不像过阻尼那么慢。

ζ > 1（过阻尼）：完全不振，但上升很慢。就像把弹簧泡在蜂蜜里——安全但迟钝。

生活类比：你推一下秋千——它会来回摆（ζ≈0 无阻尼）。 如果秋千浸在水里——摆动几次就停了（ζ<1 欠阻尼）。 如果秋千浸在胶水里——推出去后缓慢回到原位，完全不摆（ζ>1 过阻尼）。 PID 控制器的 D 参数本质上就是"人为增加阻尼"。
t04 为什么在 t05 之前？因为 PID 控制的 D 参数就是在"加阻尼"。 不理解阻尼，就不会理解为什么要加 D、加多少 D。
05

PID 控制 — 工业控制的万金油

t05_pid_control.m

PID 是什么？

PID 是 Proportional（比例）、Integral（积分）、Derivative（微分）的缩写。它是工业界使用最广泛的控制算法——估计 90% 以上的工业控制回路用的都是 PID。不是因为它"最优"，而是因为简单、好调、够用。

u(t) = Kp·e(t) + Ki·∫e(t)dt + Kd·de(t)/dt

其中 e(t) = 目标值 - 实际值 = "误差"。控制量 u 是三项的加权和。

三个参数各自的作用——逐个理解

P（比例）："现在有误差，立刻反应" — 误差大就给大力，误差小就给小力。P 越大反应越快，但太大系统会振荡甚至不稳定。只用 P 控制会有一个"稳态误差"（最终稳定值和目标值差一点点，因为当误差很小时 u=Kp·e 也很小，推不动系统了）。

I（积分）："累积误差，持续发力" — 把历史上的误差全部加起来。即使当前误差很小，只要历史上误差一直存在，积分项就会持续增大控制量，直到误差完全消除。这就是为什么需要 I——消除 P 无法消除的稳态误差。但 I 太大容易引起超调和振荡。

D（微分）："误差在变大，提前刹车" — 看误差的变化率。如果误差在快速增大（说明系统在冲过头），D 项产生一个"制动力"来抑制趋势。D 就是"增加阻尼"——减小超调，让系统更平稳。但 D 对噪声特别敏感（噪声的导数很大）。

开车类比：你要把车停到车位。
P = 看离车位还有多远，远就踩油门，近就松油门。
I = 如果车停了但离车位还差 10cm，你会一点一点往前蹭（累积的"没到位感"）。
D = 如果车速太快冲向车位，你会提前踩刹车防止冲过头。
纯 P 会在车位附近来回晃，P+D 会平稳停住但可能差一点，P+I+D 完美到位。

调参口诀

  • 先调 P：从小到大，直到响应够快但刚开始振荡
  • 再加 D：抑制 P 引起的振荡和超调
  • 最后加 I：消除残余的稳态误差
  • P 太大→振荡，I 太大→超调+慢，D 太大→对噪声敏感+控制信号抖动
理解 PID 就理解了 90% 的工业控制。后面的 LQR、MPC、H∞ 等"高级"方法都是在特定场景（多变量、有约束、有不确定性）下替代 PID 的。但 PID 永远是你最先尝试的方案。
06

信号导入导出 — 打通 MATLAB ⇄ Simulink 数据流

t06_sources_and_sinks.m

为什么需要数据导入导出？

前面几课的数据都只存在于 Simulink 内部（Scope 里看看）。但实际工作中你需要：把 MATLAB 里算好的数据喂给 Simulink 当输入 → 把仿真结果拿出来在 MATLAB 里做分析（FFT、统计、画图）。From Workspace / To Workspace 就是 Simulink 的"进出口海关"。

三个关键模块

From Workspace（数据进口） — 允许你把 MATLAB 工作区里的变量作为 Simulink 的输入信号。比如你从实验记录了真实的传感器数据，想喂给控制器模型仿真——就用这个模块。数据格式要求是 [时间向量, 数据向量]。

To Workspace（数据出口） — 把仿真结果保存到 MATLAB 工作区。仿真结束后，你可以在命令行访问这些变量，画图、做统计分析。这个是本教程批量生成图片的关键模块。

To File（保存到文件） — 把数据保存为 .mat 文件。适合长时间仿真（不想占用内存）。

类比：Simulink 是一个"仿真工厂"。From Workspace = 原材料进口， To Workspace = 成品出口，Scope = 厂里的监控屏幕。数据流 = 原料 → 加工（你的模型）→ 成品。
07

子系统封装 — 模型的层次化设计

t07_subsystem.m

问题：如果模型有 100 个模块，你怎么看得过来？

答案：把一组相关的模块"打包"成一个盒子，对外只露出输入输出端口。这就是子系统（Subsystem）。它就像编程里的"函数"——调用者不需要知道内部怎么实现的。

层次化设计的三层结构（t07 演示的）

顶层（最外面）：只有信号源 + 子系统盒子 + 示波器。极简！任何人都能看懂信号从哪来到哪去。

第二层（Signal_Processor 内部）：Gain（增益=3）+ Transfer Fcn（一阶滤波 τ=0.5）。双击盒子就能钻进去看细节。

第三层（Nested_Demo → Inner_Sub 内部）：再嵌套一层子系统，里面只有一个 Gain=4。这演示了子系统可以无限嵌套——就像代码里的函数可以调用函数。

编程类比：顶层 = main() 函数（几行代码，概括全局）。
子系统 = 子函数（封装一个功能模块）。
嵌套子系统 = 子函数调用子子函数（递归封装）。
你不需要读懂 1000 行代码——你只需要知道每个函数的"输入是什么、输出是什么"。
层次化是控制工程师的基本素养。一个电机控制系统可能有几百个模块——如果全部平铺在一个画布上，没有人能维护。 好的模型应该像洋葱：外层简洁，每剥一层看到更多细节。
08

Mask 参数化封装 — 让模块可配置、可复用

t08_masking.m

子系统 vs Mask 的区别

子系统只是"视觉折叠"，双击还是进入内部。Mask 是"参数拦截"——双击弹出一个对话框让你填参数，不用进入内部。就像一个 APP 的用户界面：你只需要填参数点确定，不用关心代码怎么写的。

Mask 三要素

  • Mask 参数（形参）：在 Mask Editor 里定义的变量，比如 myGain、myTau。这些会出现在双击弹出的对话框里。
  • 子系统内部（函数体）：模块参数引用这些变量名（比如 Gain 的值填 myGain 而不是具体数字）。
  • 模块实例（函数调用）：拖出两个同样的 Mask 模块，一个设 myGain=5，一个设 myGain=10——同一个"函数"被调用了两次，传了不同"实参"。
编程类比：定义函数 amplify(signal, gain, tau) → 这就是 Mask。
内部怎么实现的不重要，调用者只需要传 (sig, 5, 0.1) 和 (sig, 10, 2.0) 得到两个不同实例。
电机驱动器、传感器模块、控制器的 Simulink 库全都是 Masked Subsystem！
Phase 2 · 控制理论 — 从经典到现代（t09-t12）
09

状态空间模型 — 现代控制理论的大门

t09_state_space.m

传递函数 vs 状态空间

传递函数（t03-t04）：只关心"输入→输出"的关系。就像你只关心一个黑盒子的输入输出数据，不关心里面发生了什么。适合简单系统。

状态空间：把盒子拆开，看内部所有变量的流动。你不仅知道输出是什么，还知道系统"内部正在发生什么"。适合复杂系统（多输入多输出、有内部约束）。

ẋ = Ax + Bu （状态方程：状态的"变化速度"由当前状态和输入决定） y = Cx + Du （输出方程：我们能测量到的东西由状态决定）

"状态"到底是什么？

状态就是"能完全描述系统当前情况的最少一组变量"。对于质量-弹簧-阻尼系统：两个状态就够了——位移和速度。知道这两个值，你就能预测系统下一秒在哪。

状态 = 游戏存档。你玩 RPG 时，"存档"记录了角色的位置、血量、经验值、装备…… 这些就是"状态变量"。有了存档，你就能在任何时候恢复游戏状态。 状态空间模型的 A 矩阵就是"游戏规则"——描述状态如何演变。

能控性 & 能观性

能控性：能不能通过输入 u 把系统从任意初态驱动到任意终态？数学上检查 rank(ctrb(A,B)) == n。不能控的系统就像一辆只能左转不能右转的车——你到不了任何想去的地方。

能观性：能不能通过测量输出 y 反推出全部状态？数学上检查 rank(obsv(A,C)) == n。能观才可以用观测器（t11），不能观就必须加传感器。

为什么企业级应用都用状态空间？因为现实世界的系统有多个状态：
无人机 = 位置(3) + 速度(3) + 姿态角(3) + 角速度(3) = 12 个状态
机械臂 = 6 个关节 × (角度+角速度) = 12 个状态
LQR、Kalman、MPC 都必须建立在状态空间上。
10

LQR 最优控制 — 让数学帮你找最好的 K

t10_state_feedback.m

状态反馈的基本思路

如果系统不稳定或者响应太慢，怎么办？测量所有状态，乘上增益 K，反馈回去改变系统的行为。即 u = -Kx。新的闭环系统变成 ẋ = (A-BK)x——通过选择 K，我们可以把系统的极点（决定了响应速度和振荡特性）放到任何想要的位置。

极点配置 vs LQR

极点配置："我想要系统以 ζ=0.7 的阻尼、2Hz 的带宽运动"→ 直接指定极点位置 → 反算 K。适合需求明确的场景。

LQR（线性二次型调节器）："我不确定具体要多快，但我知道位置精度很重要，能耗不那么重要"→ 设权重 Q 和 R → 解 Riccati 方程 → 自动给出"最优"K。适合需求模糊但有偏好的场景。

J = ∫ (xᵀQx + uᵀRu) dt ← LQR 最小化的"代价函数"

Q 和 R 怎么调？

  • Q 大（重视状态精度）：K 会变大 → 控制更"猛" → 响应快但费电
  • R 大（重视节能）：K 会变小 → 控制更"柔" → 省电但响应慢
  • Q/R 的比例决定性能 vs 节能的平衡点——这就是 LQR 调参的全部
开车类比：极点配置 = "我要 5 秒内到 100km/h，不能超速"（精确指令）。
LQR = "我想尽快到但不想费太多油"（有偏好但没具体数字）。
实践中 LQR 更常用——大多数时候你说不清"想要多快"，但你知道"位置比速度重要 10 倍"。

状态观测器 — 当状态测不到时怎么办

t11_observer.m

问题：LQR 需要全部状态，但你可能只有一个传感器

比如你只有一个位置传感器（编码器），但 LQR 需要位置+速度两个状态。速度传感器太贵/装不下/不可靠，怎么办？用数学模型+已知信息"估计"出速度。

Luenberger 观测器的原理

ẋ̂ = Ax̂ + Bu + L(y - ŷ) ← 观测器方程 ė = (A - LC)e ← 估计误差的动态

观测器 = 系统模型预测 + 测量误差修正。它内部跑一个和真实系统一样的模型，然后不停用"模型预测输出 ŷ"和"实际测量 y"的差来修正自己。如果修正力度 L 够大，估计误差 e 就会迅速衰减到零。

类比：你蒙着眼走路。
模型预测 = "我上一秒在(0,0)，速度是 1m/s 往前 → 我现在大概在(1,0)"
测量修正 = 手摸到了墙（实际在 0.9m 处）→ 修正估计为 0.95m（信任模型也信任触觉）
L 的大小 = 你有多相信触觉——L 大 = 每摸一次墙就大幅修正，L 小 = 更相信自己的步数估计。

分离原理 — 最优美的结论之一

控制器 K 和观测器 L 可以完全独立设计！整个闭环系统的特征值 = eig(A-BK) ∪ eig(A-LC)。这意味着你可以分开调控制器（让它快）和观测器（让它更快），互不干扰。

Kalman 滤波 — 噪声世界里的最优观测器

t12_kalman_filter.m

Luenberger vs Kalman 的根本区别

Luenberger：假设没有噪声，L 靠极点配置（想要多快就设多快）。
Kalman：承认有噪声（过程噪声和测量噪声），L 靠统计优化（知道噪声大小后自动算出"最合理的 L"）。

Q 和 R 的物理含义

  • Q（过程噪声协方差）："我不信任我的模型"——模型有误差/有随机扰动。Q 越大，Kalman 越依赖测量值。
  • R（测量噪声协方差）："我不信任我的传感器"——传感器有噪声。R 越大，Kalman 越依赖模型预测。
  • Q/R 比例决定"更信模型还是更信传感器"——Kalman 自动算出最优的 L 来平衡这个 trade-off。
Q 和 R 就像一个天平的两端：
你的 GPS 很准但 IMU 很烂 → R 小 Q 大 → 信 GPS（测量）
你的 IMU 很准但 GPS 有遮挡 → R 大 Q 小 → 信 IMU（模型）
Kalman 自动算出最优权重——这就是它为什么叫"最优估计"。
工程实践：Q 和 R 是 Kalman 滤波最难调的参数。一个技巧：把 R 设为你传感器的实际噪声方差（可以直接从静止数据算出来），然后只调 Q。Q 从小到大试，直到滤波结果既平滑又能跟踪信号变化。

DC 电机控制 — 执行器建模实战

t13_dc_motor.m

为什么从电机开始？

因为电机是控制工程最常见的"执行器"——控制器算出指令，最终要电机来执行。理解电机的动态特性才能真正理解"控制器为什么需要这样设计"。

双时间尺度：电气 vs 机械

DC 电机有两个储能元件：电感 L（电能存储）和转动惯量 J（动能存储）。电气时间常数 τ_e = L/R 通常在毫秒级（≈1ms），机械时间常数 τ_m = J/b 通常在秒级（≈50s）。电气响应极快，机械响应很慢——这就是为什么需要"级联 PI"。

级联 PI 控制结构

内环（电流环 ~500Hz）：控制电流 = 控制转矩。带宽高，反应快，保护硬件（限流）。

外环（速度环 ~20Hz）：控制转速。带宽低，给内环提供电流指令。

两层 PI 串在一起，内环是外环的"执行器"。这种结构是工业伺服驱动器的标准——你的 CNC 机床、机器人关节都是这样控制的。

级联 PI 的优势：电流限幅（防止烧电机）+ 调试直观（先调内环再调外环）+ 抗负载扰动。每个电机驱动器固件里都在跑这套逻辑。

PMSM + FOC 矢量控制 — 无刷电机的核心算法

t14_foc_pmsm.m

PMSM 和 DC 电机的本质区别

DC 电机：电流天然产生转矩（换向器+电刷自动切换电流方向）。
PMSM（永磁同步电机）：三相交流电流产生旋转磁场，转子跟着转。转矩和电流的关系不直观——需要 FOC（磁场定向控制）来解耦。

FOC 的三个步骤

1. Clarke 变换：三相电流 (a,b,c) → 两相静止坐标 (α,β)。就是把三个相差 120° 的线圈电流，投影到一个二维平面上。

2. Park 变换：静止坐标 (α,β) → 旋转坐标 (d,q)。坐标系跟着转子一起转！在旋转坐标系里，电流变成了直流量（不再是正弦波）——这就可以用简单的 PI 控制了。

3. PI 控制：在 d-q 坐标系中，d 轴电流控制磁场强度（通常设 id=0），q 轴电流控制转矩（和 DC 电机完全一样！）。

类比：你在旋转木马上想接住别人扔给你的球。
从地面看（静止坐标），球的轨迹是复杂的曲线（正弦波）。
从你的视角看（旋转坐标），球就是直直地飞过来（直流量）。
Park 变换 = "站到旋转木马上看世界"——把正弦波变成直流量，控制就简单了。
FOC 是新能源车（特斯拉）、无人机（大疆）、机器人关节的核心技术。 学会 FOC = 理解无刷电机是怎么驱动的。本质还是 PI + 坐标变换！

C 代码生成 — 从 Simulink 仿真到 STM32 嵌入式

t15_code_generation.m

为什么需要"离散化"？

Simulink 跑的是连续时间仿真（用微分方程）。STM32 单片机是数字芯片，只能每隔固定时间（采样周期 Ts）执行一次计算。连续→离散就是把微分方程变成差分方程。

连续：ẋ = Ax + Bu → 离散：x[k+1] = Ad·x[k] + Bd·u[k] Ad = eA·Ts, Bd = (∫eAtdt)·B

五个步骤把 LQR 变成 C 代码

  1. 连续设计：K_c = lqr(A, B, Q, R) — 在纸上/Simulink 里设计好
  2. 离散化：sys_d = c2d(ss(A,B,C,D), Ts, 'zoh') — 把连续模型变成离散模型
  3. 离散 LQR：[K_d,~,~] = dlqr(A_d, B_d, Q, R) — 计算离散域的最优增益
  4. C 代码：u = -(K1*x1 + K2*x2) + Ki*integral — 控制律就是 3 次乘法+加法
  5. FreeRTOS 集成：把控制函数放在定时器任务里，每 Ts 秒执行一次
这套工作流就是 "基于模型的设计 (MBD)"：
物理建模 → 控制器设计 → 仿真验证 → 自动代码生成 → 硬件部署。
掌握这 5 步，你就能把任何控制算法从 Simulink 搬到真实的单片机上运行。 如果公司买了 Embedded Coder，右键点导出就能自动生成 .c/.h——省掉手写调试时间。

频域分析 — 换个角度看系统

t16_freq_domain.m

时域 vs 频域

时域：看信号的"波形"——什么时间发生什么。比如阶跃响应图。
频域：把信号拆成不同频率的正弦波之和，看系统对每个频率的反应。就像听音乐——时域是"某秒响了什么音"，频域是"低音鼓还是高音小提琴"。

Bode 图 = 控制工程师的"心电图"

Bode 图有两部分：幅频图（上）和相频图（下）。它回答两个问题：
1. 输入一个 ω 频率的正弦波，输出幅值是输入的几倍？（幅频）
2. 输出比输入滞后了多少度？（相频）

增益裕度 GM 和相位裕度 PM

GM：增益还能放大多少倍系统才不稳定。GM > 6dB 才安全。
PM：相位还能滞后多少度系统才不稳定。PM > 45° 是良好设计的标志。
这两个数越小 = 系统离不稳定越近 = 越危险。

类比：你在摇晃一座桥。
GM = 你还能用多大的力气摇它（力气太大桥会断）。
PM = 你的摇晃和桥的摆动之间还能岔开多大的角度（同步了桥就共振垮掉）。
频域分析告诉你离危险还有多远。
17

超前-滞后校正器 — 改善系统的"基因"

t17_lead_lag.m

校正器解决什么问题？

系统本身（比如一架飞机的动力学）改不了。但你可以在控制器前面串一个"校正网络"，改变整个开环的频率特性，从而让闭环表现变好。

Lead（超前）vs Lag（滞后）

Lead 超前校正：在高频段提供相位超前（正的相位），增加相位裕度 → 系统更稳定、超调更小。代价是放大高频噪声。

Lag 滞后校正：在低频段提高增益，减小稳态误差。不影响高频相位（转折频率设在很低的地方），代价是可能减慢响应。

Lead-Lag 组合：同时改善动态响应和稳态精度——工程上等价于 PI + 超前网络，是最常用的组合。

Lead: C(s) = K·(αTs+1)/(Ts+1), α>1 Lag: C(s) = K·(Ts+1)/(βTs+1), β>1
什么时候用校正器而不是 PID？PID 是通用方案，校正器是针对性方案。 当你清楚知道"我需要在这里加 30° 相位"时，校正器比 PID 更精准。 很多工业控制器内部是 PID + Lead-Lag 的组合。

系统辨识 — 不知道模型参数？从数据里猜！

t18_system_id.m

模型从哪来？

t01-t17 都假设你知道系统参数。但现实中经常不知道——设备是别人造的、参数文档丢了、系统太复杂没法手推公式。系统辨识 = 做实验 + 用数据反推模型参数。

三种辨识方法

1. 阶跃响应法（最简单）：输入阶跃信号，从响应曲线直接"读"出 K（稳态值）、τ（63.2% 时间）、θ（延迟）。适合快速估算，对噪声敏感。

2. 最小二乘法（更精确）：建立回归方程 y=Φθ，用 (ΦᵀΦ)⁻¹Φᵀy 求解最优参数 θ。有噪声时比阶跃法准得多。

3. 工具箱方法（最省事）：procest(data, 'P1D') 一行搞定。MATLAB 的 System Identification Toolbox 提供了完整的辨识→验证→迭代流程。

类比：你想知道一个黑箱子里有什么结构。
你在入口敲一下（输入），在出口听声音（输出）。
敲了 100 次，记录了 100 组"敲法+听到的声音"数据。
然后用数学方法推断箱子内部的结构——这就是系统辨识。
关键原则：永远用独立数据做验证！用训练数据辨识出来的模型，再用一组没用过的测试数据验证。 如果在训练数据上误差很小但测试数据上很大 → 过拟合了。

H∞ 鲁棒控制 — 主动考虑不确定性

t19_robust_control.m

LQR 的死穴

LQR 假设你用的模型是 100% 准确的。但现实中：弹簧刚度可能±20%、阻尼可能±30%、负载质量会变化……LQR 在这种参数波动下性能会退化甚至不稳定。H∞ 控制在设计时就考虑了"参数可能不准"这件事。

H∞ 的核心思想

H∞ 范数 = 系统从输入到输出的"最大能量增益"。H∞ 控制的目标是：在所有可能的不确定性下，这个最大增益小于 1——保证最坏情况也不差。

混合灵敏度 S/KS/T 整形

  • S = 1/(1+GK)：灵敏度——对扰动的抑制能力。低频要小（抑制扰动），高频无所谓。
  • KS = K/(1+GK)：控制灵敏度——控制能量的大小。别让控制器太"暴力"。
  • T = GK/(1+GK)：互补灵敏度——噪声抑制。高频要小（抑制传感器噪声）。
设计流程： ureal('m', 1.0, '±20%') → 选 W1/W2/W3 加权函数 → hinfsyn(P, 1, 1) → γ<1 ? 成功
H∞ vs LQR 的取舍：H∞ 名义性能略低于 LQR（保守），但全局鲁棒性强得多。 航空航天、精密运动控制——这些"不能失败"的场景必须用 H∞。 普通工业控制——PID/LQR 够用，H∞ 是过度设计。

MPC 模型预测控制 — 带约束的最优控制

t20_mpc.m

PID/LQR 的局限：不能处理约束

现实世界中：电机电压不能超过 12V、阀门开度只能在 0-100%、温度变化率不能太快……PID 和 LQR 不考虑这些约束，只能在事后"限幅"——这可能导致性能退化或振荡。

MPC 怎么工作？

每个时刻，MPC 求解一个优化问题："未来 N 步内，控制量怎么走最好？"约束条件包括输入上下限、变化率限制、输出范围……解出来后，只执行第一步，下一时刻重新优化。这叫做"滚动时域控制"。

开车类比：你开车去一个没去过的地方。
PID = 只看眼前的路（当前误差），不看前方障碍。
MPC = 看着前方 N 米的路面，规划一条避开所有坑的最优路径。
但 MPC 计算量大——你能看清 300 米（Np=300）但每算一次要很久。
所以工程上 Np 不能太大，采样周期不能太短。

MPC 的核心参数

  • Np（预测时域）：看多远。Np 越大控制越"有远见"但计算越慢。
  • Nc（控制时域）：控制几步。Nc < Np 时后面的控制量保持不变。
  • 约束：这才是 MPC 的灵魂——没有约束就用 LQR 好了，不需要 MPC。

滑模控制 SMC — 非线性鲁棒控制的"暴力美学"

t21_sliding_mode.m

滑模控制的哲学

不追求"让系统温柔地靠近目标"，而是设计一个滑模面 s(x)=0，然后用不连续的开关控制把系统"摔"到这个面上。一旦到了滑模面上，系统的行为完全由滑模面决定——和系统本身的参数无关！

s = λe + ė = 0 ← 滑模面（一阶误差动态） u = ueq + usw ueq = 等效控制（维持滑动） usw = -K·sign(s)（拽回滑模面）

抖振问题 — SMC 的代价

sign(s) 在 s=0 附近疯狂切换 → 控制信号剧烈抖动 → 执行器磨损、高频噪声。解决方案：

  • 边界层饱和函数：用 sat(s/Φ) 代替 sign(s)，Φ 是边界层厚度。Φ 大→平滑但精度降。
  • Super-Twisting 算法：二阶滑模，连续控制无抖振——高端应用的首选。
类比：你想让球停在山谷底部。
PID = 轻轻地推，慢慢调——温柔但怕风（扰动）。
SMC = 两边有墙，球只能在两墙之间的槽（滑模面）里滑动——不管风多大球都掉不出去。
代价是球一直撞墙（抖振）——加边界层 = 在墙上垫海绵（没那么震了但球能跑的范围稍大一点）。
SMC 适用场景：大扰动、参数大幅变化的系统——无人机抗风、导弹制导、电力电子变换器。 配合观测器可以做到无传感器下的高性能控制。但如果扰动不大，用 PID 就够了——SMC 的额外复杂性要有对应收益才值得。
